# 配置指南

本 Skill 需要配置少量参数才能正常工作。首次使用时会自动引导配置。

## 配置文件位置

`config.json`，位于 Skill 根目录下（与 `SKILL.md` 同级）。

## 配置项说明

```json
{
  "project_base_dir": "",
  "cost_per_day": 970
}
```

### project_base_dir（推荐）

项目资料存放的根目录。用于建议输出路径。

- 类型：字符串
- 格式：目录路径
- 示例：`"/home/user/projects/"`

**如何获取：** 确认团队统一的项目资料存放位置。

### cost_per_day（可选）

人力成本基准，单位：元/人天。

- 类型：整数
- 默认值：`970`
- 说明：用于自动计算人力费用（人天数 × cost_per_day）

## 关于模板

PPT 模板已内置在 `assets/` 目录下，按阶段分为 4 个通用文件，**无需用户配置**。脚本根据产品线自动替换公司名和承担单位。

| 模板文件 | 适用阶段 |
|---------|---------|
| `charter-template.pptx` | Charter（立项） |
| `pdcp-template.pptx` | PDCP（设计评审） |
| `adcp-template.pptx` | ADCP（结项评审） |
| `transfer-template.pptx` | 转移 |

**产品线自动替换：**

| 产品线 | 公司名 | 承担单位 | 适用范围 |
|--------|--------|---------|---------|
| hm | 四川虹美智能科技有限公司 | 数据技术所 | 空调、公司通用平台技术 |
| ml | 长虹美菱股份有限公司 | AI数智研发部 | 冰箱、洗衣机、小电日电、数字化 |

## 首次配置流程

当 Skill 检测到 `config.json` 不存在时，会引导用户完成以下步骤：

1. **确认项目目录** — 询问项目资料存放位置
2. **确认人力成本** — 确认人天单价（默认 970）
3. **保存配置** — 写入 `config.json`

## 跳过配置

如果用户只需要输出结构化要点（不需要生成 PPT 文件），可以跳过配置。Skill 仍然可以提供完整的引导式提问和内容框架。

## 更新配置

编辑 `config.json` 即可。常见更新场景：

- 项目资料目录变更 → 修改 `project_base_dir`
- 人力成本调整 → 修改 `cost_per_day`
