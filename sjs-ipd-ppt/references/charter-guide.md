# Charter（立项）阶段详细指南

## 目标

说服评审委员会：这个项目**值得做**、**应该现在做**、**投入产出合理**。

## 核心原则

Charter 不是技术方案，是**商业论证**。技术细节一笔带过，价值论证必须充分。

## 8 大模块详解

### 1. 项目背景（15%）

回答"为什么现在要做这件事"。

**内容要素：**
- 行业发展趋势（数据/案例支撑）
- 当前技术/业务现状和痛点
- 竞品/同行在做什么
- 需求来源（客户需求/技术趋势/战略规划）

**常见问题：**
- 背景描述太泛，没有具体数据支撑
- 没有说明"为什么是现在"

**好的写法：**
> 随着冰箱智能化率从 2023 年的 15% 提升至 2025 年的 42%，用户对冰箱运行状态的可视化需求显著增长。当前凝霜检测依赖人工经验，误判率约 35%，导致除霜不及时或过度除霜...

### 2. 研究内容（15%）

概述技术路线，不需要展开到实现细节。

**内容要素：**
- 核心研究方向（1-3 个）
- 技术路线概述
- 关键方法论

### 3. 项目价值（25%）— 最重要！

**价值框架：**

```
├── 技术价值：突破了什么瓶颈？建立了什么能力？
├── 经济价值：降本/增效/增收多少？投入产出比？
├── 应用价值：能用在哪些产品/场景？市场多大？
└── 战略价值：技术储备？专利布局？行业影响力？
```

**基础研究价值引导：**
- "这个技术方向目前行业做到什么程度了？"
- "如果不做，半年后/一年后会怎样？"
- "做成之后能支撑哪些上层应用？"
- "有没有可量化的技术指标提升？"

**应用研究价值引导：**
- "这个需求是谁提的？解决谁的问题？"
- "现在怎么做的？痛点是什么？"
- "项目完成后业务指标能改善多少？"
- "有对标案例或竞品吗？"

### 4. 项目目标（15%）

Q/C/D 三要素：

| 指标 | 含义 | 示例 |
|------|------|------|
| Q | 技术规格 | 凝霜识别准确率 ≥ 85% |
| C | 财务 | 总预算 XX 万元 |
| D | 进度 | 2026.03 立项 → 2026.12 验收 |

### 5. 关键资源计划（5%）

概要即可：需要几个人、什么设备、有没有外部合作。

### 6. 风险及问题管理（5%）

列出 2-3 个关键风险，给出应对措施。

### 7. 财务计划（10%）

| 费用项 | 金额 | 说明 |
|--------|------|------|
| 人力成本 | XX万 | XX人天 × 970元/人天 |
| 服务器 | XX万 | GPU服务器/云资源 |
| 试制费 | XX万 | 样机制作 |
| 其他 | XX万 | 差旅、专利等 |
| **合计** | **XX万** | |

### 8. 结论（10%）

结论部分包含三个页面：

**第九章：项目等级自评**
> "根据技术创新项目分级管理方法，对本项目等级评估如下："

| 等级 | 激励基数 | 核心要求 |
|------|---------|---------|
| A+ | 15万 | 技术突破性、行业首创/领先、专利≥3项、显著降本提效、战略价值突出 |
| A | 6万 | 技术先进性、明确业务价值、专利≥2项、可量化经济效益 |
| B | 2万 | 满足业务需求、技术可行、专利≥1项、投入产出合理 |

**第十章：项目总结**
用 2-3 句话总结 + 建议项目经理 + "恳请IPMT批准此项目的《项目授权书》，使之转入概念计划阶段。"

**专业技术委员会评审意见**（仅 A+/A 级，B 级删除此页）
